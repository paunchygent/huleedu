# 110.1: Planning Mode

## 1. Core Process
- **Analyze**: Read user query and context thoroughly
- **Research**: Consult relevant rules and documentation
- **Break Down**: Decompose complex tasks into manageable steps
- **Plan**: Create clear, numbered action steps

## 2. Planning Requirements
- **MUST** use MCP sequential thought for first prompt replies
- **MUST** consult [000-rule-index.mdc](mdc:000-rule-index.mdc) for relevant rules
- **MUST** identify architectural mandates for each step
- **MUST** include CURRENT TASK document updates

## 3. Conflict Resolution
- Identify ambiguities or rule conflicts
- Seek user clarification before implementation
- Propose alternative approaches when needed
