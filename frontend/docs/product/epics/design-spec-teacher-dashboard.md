## 1. Executive Summary

### 1.1 Purpose
Web interface for teachers to submit essays for assessment, monitor processing status, and view comparative judgment results with ML-based quality explanations.

### 1.2 Core Principles
1. **Dashboard = Command Center** â€“ Status overview and navigation hub, not a workspace
2. **Batch-Centric** â€“ Everything starts from a batch; classes/students are "friendly connections"
3. **Actionable First** â€“ Prioritize items requiring teacher action
4. **SHAP-Based Insights** â€“ Show aggregated human-readable explanations, not raw NLP metrics

### 1.3 Target User
- **Role:** Teacher (Swedish gymnasium/hÃ¶gstadium)
- **Platform:** Desktop browser (Windows primary)
- **Frequency:** Few times per week
- **Session Pattern:** Check status, confirm matches, review results

### 8.2 Essay Result

```typescript
interface EssayResult {
  id: string;
  studentName?: string;
  fileName: string;
  
  // Ranking (always present)
  cjRank: number;
  cjScore: number;              // Bradley-Terry score
  qualityBand: number;          // ML AES: 5.0, 5.5, 6.0, etc.
  cefrLevel: 'B1' | 'B2' | 'C1';

  grade?: {
    suggested: string;          // 'A', 'B', 'C/D' (uncertain), etc.
    final?: string;             // Teacher-approved grade
    confidence: 'high' | 'medium' | 'low';
    confidencePercent: number;  // 0-100
    reasoning?: string[];       // Why uncertain
  };
  
  shapFactors: ShapFactor[];
  spellingErrors: SpellingError[];
}

interface ShapFactor {
  feature: string;
  label: string;                // Teacher-facing label
  value: number;                // -1 to +1
  direction: 'positive' | 'negative';
}

interface SpellingError {
  word: string;
  suggestion: string;
  position: { line: number; column: number };
}
```
